NVIDIA GeForce RTX 3060 内核驱动安装

NVIDIA 提供了多种驱动安装方式，此处记录 NVIDIA GeForce RTX 3060 驱动的安装方法。

以下安装方式任选其一，不要混合使用。切换安装方式前，需要先卸载通过其他方式安装的 NVIDIA 驱动。

执行安装：

安装过程中的选项：

  • 在 Multiple kernel module types are available 选择 MIT/GPL，因为 RTX 3060 属于 Ampere 架构，支持 NVIDIA Open GPU Kernel Modules。

  • 在 Would you like to sign the NVIDIA kernel module? 选择 Sign the kernel module，因为系统启用了 Secure Boot，未签名的模块可能无法加载。

  • 在 Use an existing key pair or generate a new one? 选择 Generate a new key pair，因为当前没有现成的内核模块签名密钥。

  • 在 Would you like to delete the private signing key? 选择 Yes，因为本文按单次安装处理，模块完成签名后只需要保留公钥证书。

  • 在显示 X.509 公钥证书路径的页面选择 OK，并记录证书路径，后续需要将该证书导入 MOK。

  • 在 The signed kernel module failed to load 页面选择 Install signed kernel module，因为此时公钥尚未加入 MOK，完成导入和重启后即可加载。

  • 在 X Library Path 警告页面选择 OK，因为 CUDA、vLLM 和容器等无桌面计算场景通常不受影响。

  • 在 Install NVIDIA's 32-bit compatibility libraries? 选择 No，因为普通的 64 位 CUDA 和 AI 应用不需要 32 位兼容库。

  • 在 Vulkan ICD Loader 警告页面选择 OK，因为该警告不影响 CUDA、PyTorch 和 vLLM。

  • 在 Would you like to rebuild the initramfs? 选择 Rebuild initramfs，因为 initramfs 中存在 Nouveau，需要重建以避免 Nouveau 抢占 GPU。

  • 在 Would you like to run the nvidia-xconfig utility? 选择 No，因为计算节点不需要修改 X.Org 配置。
